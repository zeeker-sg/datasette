How to use this site

A practical guide for researchers, developers, and anyone who wants to explore the data available here. No jargon, just useful techniques.

{# 1. What You Can Do Here ------------------------------------ #}

What You Can Do Here

This site is a catalogue of structured data. Each record carries a summary, identifying details (titles, citations, dates, categories), and a link to the original publication. You can search and browse the catalogue, export records, and follow source links to read full documents where they were published.

1

Search Everything

Full-text search across all available data. Good for finding specific records or topics.

2

Browse by Database

Explore structured data table by table. See what is available and get a feel for the data.

3

Follow the Source

Every record links back to its original publication. Use the source link to read the full document.

4

Export Your Results

Download catalogue records as JSON or CSV for further analysis.

{# 2. What's in a record --------------------------------------- #}

What's in a Record

Catalogue records contain summaries and identifying data — not the full text of the underlying document. Full text is indexed behind the scenes so search results stay relevant, but it is not distributed here. When you find a record you care about, follow its source link to read the complete document at the original publisher.

{# 3. Common Research Tasks ----------------------------------- #}

Common Research Tasks

Here is how to tackle typical research questions using this site:

Track Developments

Goal: Stay updated on specific topics

Method: Use search with focused keywords, then bookmark the URL to check regularly.

Search for: "your topic"
Then bookmark: /search?q=your+topic

Filter and Sort a Table

Goal: Narrow a table down to the records you need

Method: Open any table, then use the column filters and sort controls. Every filtered view has its own URL you can bookmark or export.

Newest first:  /{database}/{table}?_sort_desc=date
Search a table: /{database}/{table}?_search=keyword

Find Specific References

Goal: Locate mentions of specific terms or identifiers

Method: Search with exact terms, names, or identifiers, then follow the source link on the record to read the full document.

Search for: "exact phrase"
Or: specific identifier
Or: key concept

Export for Analysis

Goal: Get catalogue records into Excel, R, Python, etc. for deeper analysis

Method: Add .csv or .json to any table URL. Exports contain the catalogue fields — summaries, identifying data, and source links.

Table data:    /{database}/{table}.csv
Filtered:      /{database}/{table}.csv?_size=1000
Search (JSON): /-/search.json?q=your+search
.json .csv
{# 4. Getting Data Out --------------------------------------- #}

Getting Data Out

Once you have found what you need, here is how to export it for further analysis:

For Excel / Google Sheets

Add .csv to any table URL. CSV files open directly in spreadsheet software. (Tables with protected full-text columns are JSON-only.)

Whole table:   /{database}/{table}.csv?_size=1000
Filtered view: /{database}/{table}.csv?_search=keyword

For Programming (Python / R / etc.)

Add .json to any table URL. JSON is easier to work with in code.

import requests
url = 'https://data.zeeker.sg/{database}/{table}.json'
data = requests.get(url).json()
{# 5. Advanced Techniques ------------------------------------ #}

Advanced Techniques

Bookmarkable Searches

Every search and filtered table view creates a unique URL you can bookmark or share.

Mobile Access

The site works on phones and tablets. Great for checking references on the go.

Automated Monitoring

Use tools like IFTTT or Zapier to monitor JSON endpoints for new data matching your criteria.

Check: /{database}/{table}.json?_search=keyword&_size=1
If row count increases, trigger alert

API Access

Every database and table has a corresponding JSON API endpoint. Add .json to the URL. See the developer docs for parameters and examples.

Table data: /{database}/{table}.json
Search:     /-/search.json?q=keyword
One record: /{database}/{table}/{id}.json
