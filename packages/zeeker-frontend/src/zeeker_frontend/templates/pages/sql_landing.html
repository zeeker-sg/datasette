What this page is for

SQL lets you ask very specific questions of the data — combine fields, filter by date ranges, count occurrences over time, group by category. It is the most expressive way to query, and also the most demanding: you need to know what tables and columns exist before you can write anything useful.

Reach for SQL when…

  • You want to count, group, or aggregate (e.g. "how many rulings per year").
  • You need to filter by more than one field at once.
  • You're combining a date range with a keyword search.
  • The built-in browse and search surfaces aren't precise enough.

Try these first if you can

  • Global search — full-text across every searchable table.
  • Browse a database — see what's there, click into a table, filter and facet.

What you can do here: read-only queries only. Each query has a 3-second time limit and returns at most 1,000 rows. Need more? Add .csv or .json to the database URL and stream it.

New to SQL? /how-to-use has worked examples you can copy.
