## Page Surfaces and Layout

### `/{db}/{table}` ŌĆö Table Browse Page

**Always-present chrome (from Phase 4 `base.html`):**
- Dark editorial nav (`.db-nav`)
- Breadcrumb strip: Home ŌĆ║ {db} ŌĆ║ {table} (`.db-crumb`)
- Hero header with italic-accent H1 + meta-col DL (`.db-header`)
- Petrol stat band: row count + column count + (page size if filtered) (`.db-statband`)
- Sticky sub-toolbar at `top: 52px` (`.db-toolbar`)

**Toolbar contents (left ŌåÆ right):**
1. FTS search input (`?_search=`) ŌĆö flex-grow, max-width 360px
2. Sort indicator (active column name + asc/desc, as text label ŌĆö no view-toggle buttons needed here)
3. Applied-facet chips (each with `├Ś` removal link) ŌĆö wraps
4. Export links (right-aligned): "CSV ŌåŚ" and "JSON ŌåŚ" as suffix-routed direct links

**Export link construction (D-05):**
```
href="/{db}/{table}.csv?{current_querystring}"
href="/{db}/{table}.json?{current_querystring}"
```
These hit Caddy's `@datasette` matcher and never enter the frontend.

**Layout modes (driven by `display.table_mode` from metadata):**

#### `table_mode: feed` (sketch 004-A)
Two-column grid: main feed list + sticky facet sidebar.
```
.feed-layout { grid-template-columns: minmax(0,1fr) 240px; gap: var(--space-12); }
```
Feed list: `<article class="va-item">` per row ŌĆö title (Fraunces 500), date (mono), category pill, excerpt (max 2 lines Inter), source host link.
Facet sidebar: sticky at `top: 120px`. Each facet column is a `.facet-block` with petrol rule-top on the first, border rule on subsequent. Facet items show label + count (mono --text-2xs). Applied ŌåÆ `.facet-item.active` (petrol bg, paper text). Up to 10 values shown; no "show all" in Phase 5 (deferred).

#### `table_mode: tabular` (default fallback, sketch 002-B base)
Standard `<table>` with `<thead>` column headers. Sortable columns: click header ŌåÆ `?_sort=col`, click again ŌåÆ `?_sort_desc=col`, click again ŌåÆ clear. Active sort column header underlined in accent. Rows use paper background; alternate rows use `--color-bg-alt` for zebra stripe.
Facets sidebar: same `.feed-layout` two-column grid as feed mode ŌĆö sidebar renders when facets available regardless of table_mode.

#### `table_mode: longform-list`
Feed list without category pills. One editorial row per record: title (Fraunces) + muted meta line. No excerpt unless `body` column slot mapped. Same two-column layout with facet sidebar.

**Pagination (all modes):**
- Datasette native cursor tokens (`next_url`, `next` from JSON payload)
- Rendered as: `ŌåÉ Previous` (when previous URL available) and `Next ŌåÆ` (when `next_url` present)
- Page-size selector: 25 / 50 / 100 as `?_size=N` links, inline with pagination row
- No numbered pages
- Pagination strip sits below the content area, above the footer

**FTS search:**
- Submits `?_search={term}` ŌĆö datasette native param
- Active search term displayed as a chip in the toolbar (dismissable with `├Ś` ŌåÆ clears `_search` from URL)
- When `_search_highlight` data present in JSON rows, apply `<mark>` with ochre background + ink text (no accent ŌĆö too heavy) on the highlighted fragment
- No results message: "No results for '{term}' in {table title}. Try broadening your search."

**Sort behavior (tabular mode):**
- Column header `<a>` links; active sort column shows Ōåæ or Ōåō Unicode arrow suffix
- Sort state carries through pagination (`next_url` from datasette already includes `_sort`)

**Empty table state:**
- Heading: "No records found"
- Body: "This table has no data yet, or all records were filtered out. Try removing filters."
- No illustration ŌĆö plain text centered in the content area.

### `/{db}/{table}/{pk}` ŌĆö Row Detail Page

**Always-present chrome:**
- Dark editorial nav
- Breadcrumb strip: Home ŌĆ║ {db} ŌĆ║ {table} ŌĆ║ {pk} (truncate pk to 12 chars + ellipsis if UUID/hash)
- Hero header (no stat band ŌĆö per skill rule: "Reserve stat band for home / database / table; row page competes with article")

**Layout modes (driven by `display.row_mode` from metadata):**

#### `row_mode: article` (sketch 003-A ŌĆö magazine variant)
Two-column article layout: `minmax(0,1fr) 280px`
- Left: kicker (mono terracotta) + H1 (italic-accent Fraunces) + byline dateline + `<div class="article-body">` with drop cap on first paragraph (when body Ōēź 3 paragraphs) + source coda (double-rule divider variant B)
- Right: sticky `.aside` panel ŌĆö `.aside-block` with petrol rule-top; `<dl>` of record metadata fields; export links (JSON / CSV direct suffix links)
- Body column: Fraunces opsz 11, 1.125rem, line-height 1.65, max-width 62ch

#### `row_mode: judgment` (sketch 003-B ŌĆö editorial broadsheet)
Full-width centered column, max-width 62ch:
- Dark dateline strip below breadcrumb (`.dateline` ŌĆö ink bg, ochre agency, mono date)
- H1 at `clamp(2.5rem, 6vw, 5rem)` Fraunces, italic-accent
- 4-column meta strip (court ┬Ę citation ┬Ę date ┬Ę judges) below title
- Article body: Fraunces opsz 11 serif
- Subject tags as chip row below body
- Source coda: double-rule `border-top: 4px double var(--color-ink)` + centered source URL + fingerprint (SHA/id) in mono muted
- No sticky sidebar (judgment content IS the experience)

#### `row_mode: longform` (sketch 003-A variant without aside)
Reading column only, max-width 62ch, no sidebar:
- Kicker + H1 + simple meta line (Inter sm, muted)
- Article body: Fraunces opsz 11
- Source link-out when `item_url` / `source_url` column present
- No drop cap (longform-guide content rarely has a distinct "first paragraph")

#### `row_mode: tabular` (default fallback)
Key-value `<dl>` of all columns:
- `<dt>` in JetBrains Mono xs uppercase muted
- `<dd>` in Inter sm ink
- Long text fields (`len > 200`) get `<details>/<summary>` expand toggle ("Show full content / Collapse")
- Export sidebar: same `.aside` panel as article mode, right-aligned at 280px
- Layout: `minmax(0,1fr) 280px` two-column grid (reading dl + aside)

**Row export links (both modes with aside):**
- "View as JSON ŌåŚ" ŌåÆ `/{db}/{table}/{pk}.json`
- "Download CSV" ŌåÆ `/{db}/{table}.csv?{pk_filter_querystring}`
Both are suffix-routed via Caddy; no frontend processing.

**Navigation within table (row detail):**
- "ŌåÉ Back to {table title}" link in breadcrumb or as secondary link below kicker
- No prev/next row navigation in Phase 5 (datasette doesn't expose this cheaply via JSON)

## Interaction Contracts

### Sort (tabular mode)
- Column header is `<a href="?_sort={col}">` when unsorted
- Active asc: `<a href="?_sort_desc={col}">` + Ōåæ suffix text
- Active desc: `<a href="/{db}/{table}">` (clear sort) + Ōåō suffix text
- Carries `_search`, `_facet`, applied facet filters through sort toggle (append to querystring)

### Facet application
- Facet item click ŌåÆ navigate to `?{col}={val}&_facet={col}` (other active facets preserved)
- Applied facet chip `├Ś` ŌåÆ navigate to URL with that facet value removed (other params preserved)
- Applied facets display as `.filter-chip` pills above the content list in the toolbar area
- Format: `{column}: {value} ├Ś` in mono xs

### FTS search
- Search input `type="search"`, submits `GET` with `?_search={term}` appended
- Active search term displayed as dismissable chip in toolbar (`├Ś` clears `_search`)
- FTS `<mark>` highlight: `background: rgba(192,138,46,0.25); color: var(--color-ink); border-radius: 2px`

### Pagination
- "ŌåÉ Previous" link: present only when `prev_url` available from datasette JSON
- "Next ŌåÆ" link: present only when `next_url` available from datasette JSON
- Page size: `?_size=25` / `?_size=50` / `?_size=100`; active size is current non-link text
- All other querystring params preserved when changing page size

### Row detail ŌĆö long text expand
- Fields with value length > 200 chars truncate to 200 chars + `<details>` wrapping rest
- `<summary>` text: "Show full content"
- Expanded `<details>` attribute: `open` renders full text, `<summary>` text changes to "Collapse" via CSS `details[open] summary { display: none }` + pseudo-content trick is NOT used ŌĆö just always render both states visible via JS toggle or native `<details>`.

### Transitions
- Feed card hover: `background: var(--color-bg-alt)` transition 0.15s ease (no translateY ŌĆö cards are list items not floating cards)
- Editorial row hover (tabular table, longform-list): left-edge slider `::before` width/height transition 0.2s ease (per skill pattern)
- Facet item hover: `background: var(--color-surface-sunken)` 0.15s ease
- No other transitions
