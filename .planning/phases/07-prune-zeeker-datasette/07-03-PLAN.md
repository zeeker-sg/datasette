<objective>
Sever the runtime S3 re-overlay path for UI assets. Without this edit, Plan 07-04's mass-delete of `templates/` and `static/` is cosmetic-only at runtime: every container restart calls `download_from_s3.py`, which checks S3 for the legacy M1 overlay and re-downloads it onto the (now-empty) container directories — defeating the entire phase. This is the load-bearing fix per 07-RESEARCH Q3 Option A.

Purpose: Make the deletion stick at runtime. The frontend now owns all HTML, so Datasette no longer needs `templates/`/`static/`/`plugins/` (other than the surviving `cache_headers.py`) downloaded from S3. The data sync (`*.db` files + the merged metadata blob the scraping pipeline writes) must remain — that is the productive job of this script.

Output: One commit edits `scripts/download_from_s3.py` — surgical edits to disable the template/static/plugins download paths and the `upload_base_assets` mirror-back loop, preserving the database-sync path verbatim.
</objective>
