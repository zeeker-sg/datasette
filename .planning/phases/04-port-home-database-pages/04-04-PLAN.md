    ```python
    """GET /{db} — database overview page. Phase 4."""
    from __future__ import annotations

    from datetime import datetime

    import httpx
    from fastapi import APIRouter, HTTPException, Request
    from fastapi.responses import HTMLResponse

    from zeeker_frontend.datasette_client import fetch_database, fetch_site_metadata

    router = APIRouter()


    @router.get("/{db}", response_class=HTMLResponse)
    async def database(request: Request, db: str):
        client: httpx.AsyncClient = request.app.state.http

        # Pitfall 1: fetch_database returns None on 404; raises on other errors.
        try:
            payload = await fetch_database(client, db)
        except httpx.HTTPError:
            raise HTTPException(status_code=503, detail="Data API unavailable")

        if payload is None:
            # Pitfall 1 revisited: explicit 404 — NOT a generic 500 traceback.
            raise HTTPException(status_code=404, detail="Database not found")

        # Pitfall 5: trust Datasette's `hidden: true` flag. It covers BOTH
        # `_zeeker_*` metadata tables AND FTS internals (*_fts, *_fts_data,
        # *_fts_docsize, *_fts_idx). Do NOT prefix-check — it misses FTS.
        visible_tables = [t for t in payload.get("tables", []) if not t.get("hidden")]

        # Per-DB title/description live in /-/metadata.json.databases[db]. Source/license
        # are already merged into /{db}.json top-level by datasette — pass both up.
        site_metadata = await fetch_site_metadata(client)
        db_entry = (site_metadata.get("databases") or {}).get(db) or {}
        merged_metadata = {
            "title": db_entry.get("title"),
            "description": db_entry.get("description"),
            "source": payload.get("source"),
            "source_url": payload.get("source_url"),
            "license": payload.get("license"),
            "license_url": payload.get("license_url"),
            # Nested tables metadata for per-table friendly titles/descriptions
            "tables": db_entry.get("tables", {}),
            # Preserve menu_links so base.html nav renders correctly
            "menu_links": site_metadata.get("menu_links", []),
        }

        breadcrumb_label = merged_metadata["title"] or db.replace("-", " ").replace("_", " ").title()

        response = request.app.state.templates.TemplateResponse(
            request=request,
            name="database.html",
            context={
                "database": db,
                "tables": visible_tables,
                "views": payload.get("views", []),
                "canned_queries": payload.get("queries", []),
                "size": payload.get("size"),
                "metadata": merged_metadata,
                "breadcrumbs": [{"label": breadcrumb_label}],
                "current_year": datetime.now().year,
            },
        )
        response.headers["Cache-Control"] = "public, max-age=60, stale-while-revalidate=300"
        return response
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → frontend `/{db}` | `db` path parameter is user-controlled input. Must not allow path traversal or arbitrary upstream fetching. |
| frontend → datasette `/{db}.json` | URL is constructed with `f"/{db}.json"` — relative to the locked `base_url`. |
| Jinja render of datasette-supplied table names/columns | Untrusted content from the /{db}.json payload (table name, column names, table description) rendered into HTML. Autoescape ON. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Tampering (SSRF) | `db: str` path parameter concatenated into httpx URL | mitigate | FastAPI/Starlette reject `/` in path params (regex `[^/]+` enforced by route matcher). `httpx.AsyncClient(base_url=...)` pins the host; relative paths cannot escape. Additionally, `..` collapses at URL parse time. Tested implicitly by Phase-3 verifier `check_negative "/sglawwatch/headlines"` (multi-segment paths don't hit `/{db}`). |
| T-04-04-02 | Tampering (XSS) | Table names / column names / descriptions rendered in template | mitigate | Jinja autoescape default-on (Starlette). Verified no explicit `\|safe` filter on dataset-supplied content in database.html (only on `s()` defaults, which are developer literals). |
| T-04-04-03 | Information Disclosure (PII leak via hidden tables) | `_zeeker_*` internal tables exposed | mitigate | Handler filters `if not t.get("hidden")` — Datasette already marks both `_zeeker_*` and FTS internals as hidden=true. Tested in `test_database_filters_hidden_zeeker_tables` and `test_database_filters_fts_tables`. |
| T-04-04-04 | Denial of Service | Hanging datasette on /{db}.json | mitigate | httpx.Timeout(10.0) set at client construction (04-01); returns 503 on timeout — never a hung worker. |
| T-04-04-05 | Information Disclosure (stack trace) | 500 error on unknown database | mitigate | `fetch_database` returns None on 404 → handler raises `HTTPException(404, detail="Database not found")` BEFORE `raise_for_status()` (RESEARCH Pitfall 1). Tested in `test_database_unknown_returns_404`. |
| T-04-04-06 | Tampering (cache poisoning) | Per-DB metadata cached via /-/metadata.json TTL cache | accept | Cache key is the global `/-/metadata.json` payload; no user-controlled input. TTL 60s bounds staleness. |
| T-04-04-07 | Elevation of Privilege (SSTI) | Jinja template injection via table description | accept | Template path is a hardcoded literal (`"database.html"`); user input never passed to `env.from_string()`. Autoescape prevents content-level escape. |
| T-04-04-08 | Information Disclosure | Datasette allow_execute_sql field leaks in /{db}.json | accept | The field is public (Datasette exposes it in every unauth response). Not rendered in HTML. |
