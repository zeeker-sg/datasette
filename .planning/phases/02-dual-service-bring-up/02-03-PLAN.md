```caddyfile
# zeeker-datasette — root Caddyfile
#
# Phase 2 of milestone M2 (Frontend / API Split).
# Role: transparent reverse proxy. 100% of traffic on :80 → the internal
# Datasette service at zeeker-datasette:8001. No suffix matchers yet;
# Phase 3 adds those by uncommenting the block below.
#
# Source pattern: caddyserver.com/docs/quick-starts/reverse-proxy
# Used by: docker-compose.yml caddy service, volume-mounted at
#   /etc/caddy/Caddyfile:ro
# DNS: relies on Compose's default project bridge network resolving
#   `zeeker-datasette` to the container IP.

{
    # Local dev / phase 2: plain HTTP only. Production overlay
    # (docker-compose.prod.yml, future) will enable auto-HTTPS at the
    # real hostname data.zeeker.sg.
    auto_https off

    # Admin endpoint (default localhost:2019) stays internal to the
    # container. docker-compose.yml does NOT publish :2019 so this is
    # only reachable via `docker compose exec caddy ...` or the
    # container's own healthcheck.
    admin localhost:2019
}

:80 {
    # Phase 2: everything → datasette, byte-identical to today's
    # direct-exposed setup.
    reverse_proxy zeeker-datasette:8001

    # Phase-3 forward-compat sketch (DO NOT uncomment in Phase 2;
    # Phase 3's plan will replace the reverse_proxy line above with
    # this three-line block):
    #
    # @datasette_api {
    #     path *.json *.csv *.db /-/*
    # }
    # reverse_proxy @datasette_api zeeker-datasette:8001
    # reverse_proxy frontend:8000
}
```

Do NOT use a hostname like `localhost { ... }` — that triggers Caddy's internal CA and browsers will warn about self-signed certs (RESEARCH Pitfall 3). Port-only site address `:80 { ... }` + `auto_https off` keeps it plain HTTP.

Do NOT publish Caddy's admin endpoint externally — plan 04 will NOT add `2019:2019` to `ports:` (RESEARCH Pitfall 6). The `admin localhost:2019` directive here keeps it bound to the loopback inside the container only.

Per D-04 (Caddy over nginx/Traefik) and D-06 (incremental migration): this file's Phase-2 shape is deliberately the smallest possible diff from "no reverse proxy" to "reverse proxy routing 100% to datasette." Phase 3 converts the `reverse_proxy zeeker-datasette:8001` line into the commented block.
